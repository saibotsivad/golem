After that, RAG (retrieval-augmented generation) is the obvious culmination — it combines §3 and §5 into a single pipeline: embed a corpus, retrieve the top-k passages relevant to a prompt, prepend them as context, then run generation. The caveat is that GPT-2 isn't instruction-tuned so it won't produce clean "answers" — it'll just continue text in the style of the context. But that's actually fine for a science demo; you can frame it honestly as "grounding generation with retrieved context" and the user can observe how the retrieved passages steer the output versus generating without them.

Beyond that, there's one more topic that stands on its own: attention visualization — extracting the attention weight matrices from GPT-2 and rendering them as a heatmap showing which tokens attend to which. This is the core mechanism that makes transformers different from prior architectures, and it's visually striking. It requires digging into the model internals a bit more (Transformers.js can return attention weights if the ONNX export includes them), but it's the most "look inside the black box" moment in the whole series.
